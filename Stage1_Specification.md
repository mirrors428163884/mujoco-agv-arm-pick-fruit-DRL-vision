# Stage 1 (Tracking) 完整规格文档

## 概述

Stage 1 是导航追踪阶段，控制移动底盘和6自由度机械臂，使末端执行器接近目标物体。采用**对称Actor-Critic**架构，可选AVP（不对称价值传播）辅助训练。

---

## 🧠 PPO 网络架构

### 架构特点

与 Stage 2 不同，Stage 1 采用**对称架构**：Actor 和 Critic 使用相同输入。

```
┌─────────────────────────────────────────────────────────────┐
│                    Observation                               │
│  ┌─────────────────┐  ┌────────────────────────────────┐   │
│  │ Vector (15维)    │  │   Depth Image (224×224) 可选   │   │
│  └────────┬────────┘  └───────────────┬────────────────┘   │
│           │                           │                     │
│           └───────────┬───────────────┘                     │
│                       ▼                                     │
│              ┌────────────────┐                             │
│              │  Actor & Critic │                            │
│              │   对称输入架构   │                            │
│              └────────────────┘                             │
└─────────────────────────────────────────────────────────────┘
```

### 网络输入 (共享)

| 组件 | 维度 | 说明 |
|------|------|------|
| Vector | 15 | 状态向量 |
| CNN Features | 256 | 深度图特征 (可选) |
| **合并后** | **271** | concat(15, 256) |

## 🟡 奖励函数 (共10项)

### 正向奖励

| 奖励项 | 范围 | 公式/条件 |
|--------|------|-----------|
| **Arm Reaching** | 0 ~ 1.0 | `1.0 * (1 - tanh(3.0 * arm_reach_dist))` (机器人坐标系) |
| **Global Reaching** | 0 ~ 0.3 | `0.3 * (1 - tanh(2.0 * ee_distance))` |
| **Base Approach** | 0 ~ 1.0 | `exp(-5 * (base_dist - 0.8)²)` 最优0.8m |
| **Orientation** | 0 ~ 2.0 | `alignment^power * 2.0` (ee < 2.0m时) |
| **Touch** | 0 ~ 10.0 | 接触时触发 |
| **Success** | 0 或 50.0 | 保持接触10步时触发 (任务完成奖励) |
| **AVP** | -5 ~ +5 | Stage 2 Critic 价值估计 (可选) |

### 负向惩罚

| 惩罚项 | 范围 | 公式/条件 |
|--------|------|-----------|
| **Impact** | ~ -4*v | `-4.0 * impact_speed` |
| **Regularization** | ~ -0.1 | `-norm(ctrl) * 0.01` |
| **Collision** | -50 或 0 | 失败终止时 (增强至-50防止"神风"行为) |
| **Plant Stem** | -0.1 ~ -2.0 | 课程学习动态调整 |
| **Plant Leaf** | ~ -0.5 | `-0.5 * (1 + ee_vel)` |
| **Action Rate** | ~ -0.5 | `-norm(Δaction) * 0.02` |

## 🏆 成功条件

### Tracking 任务

```
成功 = 末端执行器接触目标 + EE距离 < 0.25m + 保持接触 10 步
```

| 条件 | 阈值 | 说明 |
|------|------|------|
| 接触判定 | `step_touch == True` | 手掌接触目标 |
| EE距离 | < 0.25m | 确保与Stage 2启动条件匹配 |
| 持续步数 | ≥ 10 步 | 稳定接触验证 |

> **与Stage 2的衔接**: Stage 2的"grasping"阶段在`ee_distance < 0.25m`时触发。Stage 1成功时确保EE已进入此范围，可直接开始Stage 2的抓取训练。

## 🔗 AVP (不对称价值传播)

Stage 1 使用预训练的 Stage 2 Critic 提供额外的"可抓取性"奖励信号。

### AVP 配置

| 参数 | 值 | 说明 |
|------|-----|------|
| `enabled` | True | 主开关 |
| `lambda_start` | 0.8 | 训练初期权重 |
| `lambda_end` | 0.2 | 训练后期权重 |
| `gate_distance` | 1.5m | 距离门控 |

### AVP 工作原理

1. 构建虚拟观测 (假设已到达抓取就位姿态)
2. 使用 Stage 2 Critic 估计价值
3. 将价值乘以 λ 作为额外奖励

```python
avp_reward = λ * clip(critic_value, -5, +5)
```
