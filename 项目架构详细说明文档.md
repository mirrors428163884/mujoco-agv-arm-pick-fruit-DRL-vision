# Picking_Sorting 项目架构详细说明文档

> **最后更新**: 2025-01-04 (重大重构版本)

## 📋 项目概述

本项目是一个基于**两阶段深度强化学习**的农业采摘机器人控制系统，使用 **AVP (Asymmetric Value Propagation)** 技术连接两个训练阶段。

### 训练顺序

```
Stage 2 (灵巧手抓取) → 导出Critic → Stage 1 (底盘+机械臂接近)
```

> **原因**: Stage 1 需要加载预训练的 Stage 2 Critic 来计算 AVP 奖励，引导机器人到达"便于抓取"的位置。

### [2025-01-04] 重大更新摘要

1. **奖励函数重构**: "Progress + Terminal + Regularization" 三阶段结构
2. **Stage 1 成功判定**: 基于预抓取姿态，**不需要接触**
3. **Stage 2 三阶段课程**: 基础抓取→滑移惩罚→扰动测试
4. **AVP改进**: 势能整形、OOD检测、自适应Lambda调度
5. **网络架构**: 分离输出头、可选GRU记忆模块
6. **域随机化**: YOLO检测帧丢失模拟

## 🎯 核心技术详解

### 1. AVP (Asymmetric Value Propagation)

#### 问题背景

Stage 1 仅靠"距离奖励"无法评估当前位置是否便于后续抓取。

#### 解决方案

使用预训练的 Stage 2 Critic 评估"可抓取性"作为辅助奖励信号。

#### [2025-01-04 重大更新] AVP改进

```
                   Stage 1 训练流程 (改进版)
                        │
          ┌─────────────┴─────────────┐
          │                           │
      原始奖励                      AVP奖励
      (Progress-based,              (势能整形)
       Terminal, Regularization)        │
          │                     ┌─────┴─────┐
          │                 构造虚拟观测 (改进):
          │                 [真实手臂状态/EE姿态]
          │                 [真实EE速度 (clip±0.5)]
          │                 [真实物体位置]
          │                 [真实深度图 (带噪声)]
          │                 [虚拟手部近抓取姿态]
          │                     │
          │               [OOD检测]
          │               - 视觉有效性检查 (60%阈值)
          │               - MC Dropout不确定性估计
          │                     │ (低置信度时跳过)
          │                     ▼
          │               Stage 2 Critic → Φ(s')
          │                     │
          │               势能整形:
          │               r_avp = λ × (γ·Φ(s') - Φ(s))
          │               clip到 [-0.2, 0.2]
          │                     │
          └──────────┬──────────┘
                     │
             总奖励 = 原始奖励 + AVP奖励
```

### 3. 奖励函数详解 [2025-01-04 重大重构]

#### Stage 1 奖励 (`RewardManagerStage1.py`)

**[NEW] Progress + Terminal + Regularization 三阶段结构:**

| 类别 | 奖励分量 | 计算公式 | 权重 | 说明 |
|------|----------|----------|------|------|
| **Progress** | EE进度 | `k × (d_prev - d_curr)`, clip±0.2 | 3.0 | 接近目标获得正奖励 |
| **Progress** | 底盘进度 | `k × (err_prev - err_curr)`, clip±0.2 | 2.0 | 接近最优距离(0.8m) |
| **Regularization** | 存活惩罚 | `-0.01` per step | -0.01 | 防止原地不动 |
| **Regularization** | 停滞惩罚 | `-0.1` if N步无进展 | -0.1 | EE距离无变化时惩罚 |
| **Conditional** | 朝向奖励 | `k × max(0, cos(θ))`, 仅d_ee<0.3m | 0.4 | 基于余弦的朝向奖励 |
| **Terminal** | 预抓取成功 | `+50` | 50.0 | **不需要接触**, 基于姿态 |
| **Terminal** | 碰撞惩罚 | `-50` | -50.0 | 严重碰撞 |
| **Terminal** | 超时惩罚 | `-20` | -20.0 | Episode超时 |
| **Optional** | AVP奖励 | `λ × (γ·Φ(s') - Φ(s))`, clip±0.2 | 自适应 | 势能整形AVP |
| **Optional** | 触摸奖励 | `10 - 4 × impact_speed` | 10.0 | 轻触(非必需) |

**[NEW] 预抓取成功条件 (不需要接触):**
```python
def check_pregrasp_pose(info):
    # 所有条件必须同时满足
    return (
        info["ee_distance"] < 0.05 and       # d_ee < 5cm
        0.7 < info["base_distance"] < 0.9 and # 底盘在最优范围
        cos(angle_error) > 0.966 and          # 角度 < 15°
        ee_speed < 0.05                        # 速度 < 5cm/s
    )
```

#### Stage 2 奖励 (`RewardManagerStage2.py`)

**[NEW] 抓取质量 + 滑移/冲击惩罚 结构:**

| 类别 | 奖励分量 | 计算公式 | 权重 | 说明 |
|------|----------|----------|------|------|
| **Progress** | EE进度 | `k × (d_prev - d_curr)`, clip±0.2 | 3.0 | 基于进度的距离奖励 |
| **Progress** | 里程碑 | `+2` if d_ee < 0.05m | 2.0 | 仅保留一个关键里程碑 |
| **Grasp** | 多指计数 | `k × (n_contact / 4)` | 1.5 | 鼓励更多手指接触 |
| **Grasp** | 力范围 | 力在[f_low, f_high]范围内 | 1.0 | 鼓励适当抓取力 |
| **Grasp** | 力平衡 | `-k × variance(forces)` | 0.5 | 惩罚力分布不均 |
| **Grasp** | 手指协同 | 拇指+其他对握 | 1.0 | 鼓励对握姿态 |
| **Penalty** | 滑移惩罚 | `-k_s × |v_rel|` | 课程自适应 | 惩罚物体-手相对速度 |
| **Penalty** | 冲击惩罚 | `-penalty if v>0.3 at first contact` | -5.0 | 惩罚高速首次接触 |
| **Terminal** | 成功奖励 | `+100` | 100.0 | 稳定抓取1秒 |
| **Terminal** | 失败惩罚 | `-50` | -50.0 | 成功/失败差距大 |
| **Perturbation** | 扰动测试 | `+10 / -5` | ±10.0 | 抵抗随机力 |

**[NEW] Stage 2 三阶段课程:**

| 阶段 | 步数范围 | 主要目标 | 配置 |
|------|----------|----------|------|
| **Phase 0** | 0-2M | 学习基础抓取 | 弱DR、近距离初始化(3-6cm)、无扰动 |
| **Phase 1** | 2M-8M | 学习稳定抓取 | 滑移惩罚渐增(0.2→1.0)、更大初始化范围 |
| **Phase 2** | 8M+ | 学习扰动抵抗 | 成功率>60%时激活扰动测试 |

#### 课程更新逻辑

```python
def update_curriculum_difficulty(self):
    # 计算课程进度 (0.0 → 1.0)
    difficulty = min(global_step / stage1_steps, 1.0)
    
    # 更新茎秆碰撞惩罚
    w_stem = w_stem_start + (w_stem_end - w_stem_start) * difficulty
    # -0.1 → -2.0
    
    # 更新朝向严格度
    orient_power = power_start + (power_end - power_start) * difficulty
    # 1.0 → 1.5
```

## 🔑 关键技术总结

1. **两阶段分离**: Stage 1/2 独立文件结构，便于调试和维护
2. **AVP连接**: Stage 2 Critic 为 Stage 1 提供抓取可行性信号
3. **动态课程**: 碰撞惩罚和朝向要求随训练进度渐进
4. **关节空间控制**: 直接输出6维关节增量，避免IK不稳定
5. **虚拟观测**: AVP使用固定手臂姿态 + 真实物体位置 + 真实深度
6. **域随机化**: 深度噪声 + 光照 + 物体 + PID参数随机化
7. **触觉反馈**: 4指触觉传感器用于抓取质量评估
8. **扰动测试**: 随机外力测试抓取稳定性
