# Picking_Sorting 项目架构详细说明文档

## 📋 项目概述

本项目是一个基于**两阶段深度强化学习**的农业采摘机器人控制系统，使用 **AVP (Asymmetric Value Propagation)** 技术连接两个训练阶段。

### 训练顺序

```
Stage 2 (灵巧手抓取) → 导出Critic → Stage 1 (底盘+机械臂接近)
```

> **原因**: Stage 1 需要加载预训练的 Stage 2 Critic 来计算 AVP 奖励，引导机器人到达"便于抓取"的位置。

## 🎯 核心技术详解

### 1. AVP (Asymmetric Value Propagation)

#### 问题背景

Stage 1 仅靠"距离奖励"无法评估当前位置是否便于后续抓取。

#### 解决方案

使用预训练的 Stage 2 Critic 评估"可抓取性"作为辅助奖励信号。

```
                   Stage 1 训练流程
                        │
          ┌─────────────┴─────────────┐
          │                           │
      原始奖励                      AVP奖励
      (距离、朝向、碰撞等)             │
          │                     ┌─────┴─────┐
          │                 构造虚拟观测:
          │                 [ready_pose手臂]
          │                 [真实物体位置]
          │                 [真实深度图像]
          │                 [虚拟手部张开]
          │                     │
          │               Stage 2 Critic
          │               (预训练, 冻结)
          │                     │
          │               value_estimate
          │                     │
          └──────────┬──────────┘
                     │
             总奖励 = 原始奖励 + λ(t) × AVP奖励
```

#### AVP关键代码位置

| 文件 | 函数/类 | 功能 |
|------|---------|------|
| `configs/env/DcmmCfg.py` | `class avp` | AVP配置参数 |
| `RewardManagerStage1.py` | `_init_avp()` | 初始化AVP组件 |
| `RewardManagerStage1.py` | `_load_stage2_critic()` | 加载Stage 2 Critic |
| `RewardManagerStage1.py` | `_construct_virtual_obs()` | 构造虚拟观测 |
| `RewardManagerStage1.py` | `compute_avp_reward()` | 计算AVP奖励 |

#### AVP配置参数

```python
class avp:
    enabled = True                    # 主开关
    lambda_weight_start = 0.8         # 早期训练权重 (强引导)
    lambda_weight_end = 0.2           # 后期训练权重 (弱引导)
    gate_distance = 1.5               # 距离门限 (仅在此距离内计算)
    checkpoint_path = "assets/checkpoints/avp/stage2_critic.pth"
    
    # 虚拟观测配置
    ready_pose = np.array([0.0, 0.0, 0.0, 1.8, 0.0, -0.785])  # 预抓取姿态
    state_dim = 35                    # Stage 2 状态维度
    img_size = 84                     # 深度图尺寸
```

### 3. 奖励函数详解

#### Stage 1 奖励 (`RewardManagerStage1.py`)

| 奖励分量 | 计算公式 | 权重 | 说明 |
|----------|----------|------|------|
| **手臂到达** | `2.0 × (1 - tanh(3 × d_arm))` | 2.0 | 鼓励机械臂末端靠近目标 (基座坐标系) |
| **全局到达** | `0.5 × (1 - tanh(2 × d_ee))` | 0.5 | 鼓励末端靠近目标 (世界坐标系) |
| **底盘定位** | `exp(-5 × (d_base - 0.8)²)` | 1.0 | 鼓励底盘保持0.8m最优距离 |
| **手臂运动** | `0.5 × tanh(3 × joint_deviation)` | 0.5 | 奖励手臂关节偏离初始位置 |
| **手臂动作** | `0.2 × ‖arm_action‖` | 0.2 | 奖励使用手臂控制 |
| **朝向对齐** | `max(0, alignment)^power × 2` | 动态 | power从1.0→1.5渐进 |
| **接触奖励** | `10 - 4 × impact_speed` | 10.0 | 轻触目标，惩罚高速撞击 |
| **树干碰撞** | `w_stem` | -0.1→-2.0 | 课程学习渐进惩罚 |
| **树叶碰撞** | `-0.5 × (1 + velocity)` | -0.5 | 速度相关轻微惩罚 |
| **动作平滑** | `-0.02 × ‖Δaction‖` | -0.02 | 惩罚动作剧烈变化 |
| **控制正则** | `-0.005×‖base‖ - 0.001×‖arm‖` | 动态 | 底盘惩罚>手臂惩罚 |
| **AVP奖励** | `λ(t) × clip(critic_value, -5, 5)` | 0.2→0.8 | Stage 2 Critic值估计 |

**AVP权重衰减:**
```python
# λ(t) 随课程进度衰减
λ = λ_start + (λ_end - λ_start) × difficulty
# difficulty = min(global_step / stage1_steps, 1.0)
```

#### Stage 2 奖励 (`RewardManagerStage2.py`)

| 奖励分量 | 计算公式 | 权重 | 说明 |
|----------|----------|------|------|
| **到达奖励** | `2.0 × exp(-2 × d_ee)` | 2.0 | 指数衰减到达奖励 |
| **距离里程碑** | 累加bonus | +5.0 | d<0.30m (+0.5), d<0.15m (+1.0), d<0.08m (+1.5), d<0.05m (+2.0) |
| **朝向奖励** | `max(0, alignment) × 1.0` | 1.0 | 仅d<0.5m时计算 |
| **抓取奖励** | `1.0 + 0.5×fingers + force_bonus` | 动态 | fingers=触觉激活数量 |
| **力范围bonus** | `+2.0` (0.5N≤F≤3.0N) | 2.0 | 理想抓取力范围 |
| **扰动测试** | `+10.0 / -5.0` | ±10.0 | 0.5-1.5N随机力，滑移<1cm通过 |
| **冲击惩罚** | `-min(2×(v-0.3), 3)` | -3.0 | v>0.3m/s时激活 |
| **成功奖励** | `+20.0` | 20.0 | 稳定抓取≥1秒 |
| **碰撞惩罚** | `-2.0` | -2.0 | 非成功终止 |
| **植物碰撞** | `w_stem / -0.05` | 动态 | 茎秆>叶片 |

**成功条件:**
```python
# Stage 2 成功判定
if (fingers_with_stable_contact >= 2 and 
    0.1N <= force_per_finger <= 2.0N and
    stable_touch_timer >= 1.0s and
    perturbation_test_passed):
    success = True
```

### 5. 课程学习详解

#### 课程参数 (`DcmmCfg.curriculum`)

```python
class curriculum:
    # 课程长度
    stage1_steps = 2e6      # Stage 1: 2M步完成课程
    stage2_steps = 10e6     # Stage 2: 10M步扩展课程
    
    # 碰撞惩罚渐进
    collision_stem_start = -0.1   # 初始轻微惩罚
    collision_stem_end = -2.0     # 最终严厉惩罚
    
    # 朝向要求渐进
    orient_power_start = 1.0      # 初始线性
    orient_power_end = 1.5        # 最终1.5次方
    
    # Stage 2 两阶段训练
    phase1_steps = 15e6     # Phase 1: Actor + Critic 联合训练
    phase2_steps = 10e6     # Phase 2: 仅Critic训练
    
    # 自适应课程
    success_rate_threshold = 0.30
```

#### 课程更新逻辑

```python
def update_curriculum_difficulty(self):
    # 计算课程进度 (0.0 → 1.0)
    difficulty = min(global_step / stage1_steps, 1.0)
    
    # 更新茎秆碰撞惩罚
    w_stem = w_stem_start + (w_stem_end - w_stem_start) * difficulty
    # -0.1 → -2.0
    
    # 更新朝向严格度
    orient_power = power_start + (power_end - power_start) * difficulty
    # 1.0 → 1.5
```

### 6. 域随机化详解

#### 深度噪声 (`DcmmCfg.depth_noise`)

模拟RealSense D435i在户外农业场景的噪声特性:

```python
class depth_noise:
    enabled = True
    
    # Cutout噪声 (模拟深度空洞)
    cutout_enabled = True
    cutout_num = (1, 3)          # 1-3个矩形遮挡
    cutout_size = (0.05, 0.15)   # 占图像5%-15%
    
    # Salt-Pepper噪声 (模拟镜面反射)

    # 边缘模糊
```

## 🔑 关键技术总结

1. **两阶段分离**: Stage 1/2 独立文件结构，便于调试和维护
2. **AVP连接**: Stage 2 Critic 为 Stage 1 提供抓取可行性信号
3. **动态课程**: 碰撞惩罚和朝向要求随训练进度渐进
4. **关节空间控制**: 直接输出6维关节增量，避免IK不稳定
5. **虚拟观测**: AVP使用固定手臂姿态 + 真实物体位置 + 真实深度
6. **域随机化**: 深度噪声 + 光照 + 物体 + PID参数随机化
7. **触觉反馈**: 4指触觉传感器用于抓取质量评估
8. **扰动测试**: 随机外力测试抓取稳定性
